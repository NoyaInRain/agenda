2. Web API¶

2.1. Introduction¶

Meetling provides a JSON REST API.

Arguments are passed to an endpoint simply as JSON object and the result is returned as JSON value. Objects contain a __type__ attribute that holds the name of the object type.

If a requested endpoint doesn’t exist, a NotFoundError is returned. For any endpoint, an InputError is returned if the input contains invalid arguments.

2.2. Authentication and Permissions¶

To make an API request authenticated as some user, include a cookie named auth_secret with the User ‘s auth_secret.

If user authentication with the given secret fails, an AuthenticationError is returned. For any endpoint, a PermissionError is returned if the current user is not allowed to perform the action.

2.3. Meetling¶

Meetling application.

POST /api/login¶

{"code": null}

Log in an User (device) and return them.

If code is given, log in an existing user with the login code. If the login fails, a ValueError (code_invalid) is returned.

If code is null, create and log in a new user. The very first user who logs in is registered as staff member.

POST /api/meetings¶

{"title", "time": null, "location": null, "description": null}

Create a Meeting and return it.

Permission: Authenticated users.

POST /api/create-example-meeting¶

Create a Meeting with an example agenda and return it.

Useful to illustrate how meetings work.

Permission: Authenticated users.

2.4. Object¶

Object in the Meetling universe.

id¶

Unique ID of the object.

trashed¶

Indicates if the object has been trashed (deleted).

2.5. Editable¶

Object that can be edited.

The URL that uniquely identifies an object is referred to as object-url, e.g. meetings/abc for a Meeting with the id abc.

authors

User s who edited the object.

POST /api/(object-url)¶

{attrs...}

Edit the attributes given by attrs and return the updated object.

A trashed (deleted) object cannot be edited. In this case a ValueError (object_trashed) is returned.

Permission: Authenticated users.

2.6. User¶

Meetling user.

User is an Object and Editable by the user oneself.

name

Name or nick name.

auth_secret

Secret for authentication. Visible only to the user oneself.

GET /api/users/(id)¶

Get the user given by id.

2.7. Settings¶

App settings.

Settings is an Object and Editable by staff members.

title

Site title.

icon

URL of the site icon. May be null.

favicon

URL of the site icon optimized for a small size. May be null.

staff

Staff users.

GET /api/settings¶

Get the settings.

2.8. Meeting¶

Meeting.

Meeting is an Object and Editable.

title

Title of the meeting.

time

Date and time the meeting begins. May be null.

location

Location where the meeting takes place. May be null.

description

Description of the meeting. May be null.

GET /api/meetings/(id)¶

Get the meeting given by id.

GET /api/meetings/(id)/items¶

Get the list of AgendaItem s on the meeting’s agenda.

If /trashed is appended to the URL, only trashed (deleted) items are returned.

POST /api/meetings/(id)/items¶

{"title", "duration": null, "description": null}

Create an AgendaItem and return it.

Permission: Authenticated users.

POST /api/meetings/(id)/trash-agenda-item¶

{"item_id"}

Trash (delete) the AgendaItem with item_id.

If there is no item with item_id for the meeting, a ValueError (item_not_found) is returned.

Permission: Authenticated users.

POST /api/meetings/(id)/restore-agenda-item¶

{"item_id"}

Restore the previously trashed (deleted) AgendaItem with item_id.

If there is no trashed item with item_id for the meeting, a ValueError (item_not_found) is returned.

Permission: Authenticated users.

POST /api/meetings/(id)/move-agenda-item¶

{"item_id", "to_id"}

Move the AgendaItem with item_id to the position directly after the item with to_id.

If to_id is null, move the item to the top of the agenda.

If there is no item with item_id or to_id for the meeting, a ValueError (item_not_found or to_not_found) is returned.

Permission: Authenticated users.

2.9. AgendaItem¶

Item on a Meeting ‘s agenda.

AgendaItem is an Object and Editable.

title

Title of the item.

duration

Time the agenda item takes in minutes. May be null.

description

Description of the item. May be null.

GET /api/meetings/(meeting-id)/items/(item-id)¶

Get the item given by item-id.

2.10. ValueError¶

Returned for value-related errors.

code¶

Error string providing more information about the problem.

2.11. InputError¶

Returned if the input to an endpoint contains one or more arguments with an invalid value.

InputError is a ValueError with code set to input_invalid.

errors¶

Map of argument names / error strings for every problematic argument of the input.

2.12. NotFoundError¶

Returned if a requested endpoint does not exist.

2.13. AuthenticationError¶

Returned if user authentication fails.

2.14. PermissionError¶

Returned if the current user is not allowed to perform an action.
