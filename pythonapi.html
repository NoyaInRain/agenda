3. Python API¶

3.1. micro¶

Toolkit for social micro web apps.

micro is based on Redis and thus any method may raise a RedisError if there is a problem communicating with the Redis server.

class micro.Application(redis_url='', email='bot@localhost', smtp_url='', render_email_auth_message=None)¶

See Application.

user¶

Current User. None means anonymous access.

users¶

Map of all User s.

redis_url¶

See --redis-url command line option.

email¶

Sender email address to use for outgoing email. Defaults to bot@localhost.

smtp_url¶

See --smtp-url command line option.

render_email_auth_message¶

Hook function of the form render_email_auth_message(email, auth_request, auth), responsible for rendering an email message for the authentication request auth_request. email is the email address to authenticate and auth is the secret authentication code.

r¶

Redis database. More precisely a JSONRedis instance.

authenticate(secret)¶

Authenticate an User (device) with secret.

The identified user is set as current user and returned. If the authentication fails, an AuthenticationError is raised.

get_object(id, default=<class 'KeyError'>)¶

Get the Object given by id.

default is the value to return if no object with id is found. If it is an Exception, it is raised instead.

login(code=None)¶

See POST /api/login.

The logged-in user is set as current user.

settings¶

App Settings.

update()¶

Update the database.

If the database is fresh, it will be initialized. If the database is already up-to-date, nothing will be done. It is thus safe to call update() without knowing if an update is necessary or not.

Subclass API: Must be implemented by subclass.

class micro.Object(id, trashed, app)¶

See Object.

app¶

Context Application.

json(restricted=False, attrs={})¶

Return a JSON object representation of the object.

The name of the object type is included as __type__.

By default, all attributes are included. If restricted is True, a restricted view of the object is returned, i.e. attributes that should not be available to the current Meetling.user are excluded.

Subclass API: May be overridden by subclass. The default implementation returns the attributes of Object. restricted is ignored.

class micro.Editable(authors)¶

See Editable.

The Object.json() method of editable objects accepts an additional argument include_users. If it is True, User s are included as JSON objects (instead of IDs).

do_edit(**attrs)¶

Subclass API: Perform the edit operation.

More precisely, validate and then set the given attrs.

Must be overridden by host. Called by edit(), which takes care of basic permission checking, managing authors and storing the updated object in the database.

edit(**attrs)¶

See POST /api/(object-url).

json(restricted=False, include_users=False)¶

Subclass API: Return a JSON object representation of the editable part of the object.

class micro.User(id, trashed, app, authors, name, email, auth_secret)¶

See User.

finish_set_email(auth_request, auth)¶

See POST /api/users/(id)/finish-set-email.

json(restricted=False, include_users=False)¶

See Object.json().

remove_email()¶

See POST /api/users/(id)/remove-email.

send_email(msg)¶

Send an email message to the user.

msg is the message string of the following form: It starts with a line containing the subject prefixed with ``Subject: ``, followed by a blank line, followed by the body.

If the user’s :email is not set, a :exception:`ValueError` (user_no_email) is raised. If communication with the SMTP server fails, an EmailError is raised.

set_email(email)¶

See POST /api/users/(id)/set-email.

store_email(email)¶

Update the user’s email address.

If email is already associated with another user, a ValueError (email_duplicate) is raised.

class micro.Settings(id, trashed, app, authors, title, icon, favicon, staff)¶

See Settings.

exception micro.ValueError¶

See ValueError.

The first item of args is also available as code.

exception micro.InputError(errors={})¶

See InputError.

To raise an InputError, apply the following pattern:

def meow(volume):
    e = InputError()
    if not 0 < volume <= 1:
        e.errors['volume'] = 'out_of_range'
    e.trigger()
    # ...
trigger()¶

Trigger the error, i.e. raise it if any errors are present.

If errors is empty, do nothing.

exception micro.PermissionError¶

See PermissionError.

3.2. micro.util¶

Various utilities.

micro.util.str_or_none(str)¶

Return str unmodified if it has content, otherwise return None.

A string is considered to have content if it contains at least one non-whitespace character.

micro.util.randstr(length=16, charset='abcdefghijklmnopqrstuvwxyz')¶

Generate a random string.

The string will have the given length and consist of characters from charset.

micro.util.parse_isotime(isotime)¶

Parse an ISO 8601 time string into a naive datetime.datetime.

Note that this rudimentary parser makes bold assumptions about the format: The first six components are always interpreted as year, month, day and optionally hour, minute and second. Everything else, i.e. microsecond and time zone information, is ignored.

micro.util.check_email(email)¶

Check the email address.
